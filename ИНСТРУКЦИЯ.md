# Инструкция

## Что нужно скачать заранее

Нужно поставить только две вещи:

1. `Python 3.11` или новее
2. `Node.js LTS`

Скачивать лучше с официальных сайтов:

- Python: [https://www.python.org/downloads/windows/](https://www.python.org/downloads/windows/)
- Node.js: [https://nodejs.org/](https://nodejs.org/)

## Что не нужно скачивать вручную

Это приложение само подтянет нужные вещи при первом запуске:

- `ffmpeg`
- Python-зависимости для backend
- npm-зависимости для frontend

## Перед установкой

Когда ставишь Python, обязательно включи галочку:

`Add Python to PATH`

Если эту галочку не включить, запуск может не сработать.

## Самый простой запуск

Если нужен самый простой способ без лишней возни:

1. Открой папку проекта
2. Дважды кликни по [start-app.bat](./start-app.bat)
3. Подожди

Что сделает этот файл:

- создаст `.venv`, если её ещё нет
- установит backend-зависимости
- скачает `ffmpeg`
- установит frontend-зависимости
- откроет backend и frontend в двух отдельных окнах

После этого открывай:

[http://127.0.0.1:5173](http://127.0.0.1:5173)

## Если запуск идёт долго

Это нормально на первом старте.

Первый запуск может идти несколько минут, потому что проект:

- ставит тяжёлые Python-пакеты
- ставит `torch`, `demucs`, `faster-whisper`
- скачивает `ffmpeg`

Первая обработка трека тоже может быть долгой. Следующие запуски обычно быстрее.

## Если хочешь запускать вручную

### 1. Подготовить backend

Открой PowerShell в папке проекта и выполни:

```powershell
.\backend\bootstrap.ps1
```

### 2. Подготовить frontend

```powershell
cd .\frontend
cmd /c npm.cmd install
cd ..
```

### 3. Запустить backend

```powershell
.\scripts\run-backend.bat
```

### 4. Запустить frontend

В новом окне:

```powershell
.\scripts\run-frontend.bat
```

### 5. Открыть приложение

Открой в браузере:

[http://127.0.0.1:5173](http://127.0.0.1:5173)

## Очень важный момент

Backend нельзя запускать так:

```powershell
python -m uvicorn backend.app.main:app
```

Почему:

- Windows может взять не тот Python
- тогда backend стартует, но потом будет падать на обработке
- типичная ошибка в этом случае: `demucs is not installed`

Правильный способ:

- [start-app.bat](./start-app.bat)
- [run-backend.bat](./scripts/run-backend.bat)

## Если что-то не запускается

### Ошибка `Python was not found in PATH`

Значит Python не установлен или не добавлен в `PATH`.

Что делать:

1. Переустановить Python
2. Включить галочку `Add Python to PATH`
3. Открыть новое окно терминала
4. Запустить снова

### Ошибка `npm` или `npx` not found

Значит не установлен Node.js.

Что делать:

1. Установить `Node.js LTS`
2. Закрыть терминал
3. Открыть его заново
4. Повторить запуск

### Ошибка про порт `8000`

Это значит, что backend уже запущен или порт занят другой программой.

Что делать:

- если backend уже работает, просто оставь как есть
- если порт занят чем-то чужим, закрой ту программу и запусти backend снова

### Окно пишет `Backend already running on http://127.0.0.1:8000`

Это не ошибка.

Это значит:

- backend уже работает
- второй раз запускать его не нужно

### В приложении видно `Pipeline failed`

Сначала проверь, чем запущен backend.

Если запуск был не через [start-app.bat](./start-app.bat) или [run-backend.bat](./scripts/run-backend.bat), проблема почти наверняка в этом.

## Что желательно иметь на компьютере

Минимально:

- стабильный интернет для первого запуска
- несколько гигабайт свободного места
- обычный современный процессор

Если есть видеокарта `NVIDIA`, обработка может идти быстрее, но приложение умеет работать и без неё.
