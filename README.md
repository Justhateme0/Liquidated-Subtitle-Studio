# Liquidated Subtitle Studio

Локальное Windows-приложение для создания субтитров из трека:

- загрузка песни
- выделение вокала через `Demucs`
- распознавание речи через `faster-whisper`
- ручная правка текста и таймингов
- экспорт в `alpha MOV` и `MP4`

## Что это такое

Это не облачный сервис и не Telegram-бот.
Приложение запускается у тебя на компьютере и обрабатывает аудио локально.

Подходит для:

- эдитов
- клипов
- субтитров в стиле `brat`
- роликов с собственным фоном, шрифтом и ручной правкой строк

## Стек

- Frontend: `React 19 + TypeScript + Vite`
- Backend: `FastAPI`
- Распознавание: `faster-whisper`
- Разделение вокала: `Demucs`
- Рендер: `ffmpeg`

## Быстрый запуск

Нужно заранее установить только:

1. `Python 3.11+`
2. `Node.js LTS`

После этого:

1. Скачай проект
2. Открой папку проекта
3. Запусти [`start-app.bat`](./start-app.bat)
4. Открой [http://127.0.0.1:5173](http://127.0.0.1:5173)

## Важное предупреждение про интернет

Будь готов к тому, что на первом запуске и на первом распознавании песни приложению может понадобиться интернет.

Почему:

- при первом запуске ставятся зависимости
- могут докачиваться модели для `Demucs` и `faster-whisper`
- из-за этого первый прогон иногда идёт заметно дольше

Если интернет в этот момент пропадёт или будет очень нестабильным, этап обработки или распознавания речи может сорваться.

Проще говоря:

- первый запуск лучше делать со стабильным интернетом
- если распознавание внезапно упало на первом прогоне, часто помогает просто повторить запуск позже

## Как запускать правильно

Backend нужно запускать только так:

- через [`start-app.bat`](./start-app.bat)
- или через [`scripts/run-backend.bat`](./scripts/run-backend.bat)

Не запускай backend через обычный:

```powershell
python -m uvicorn backend.app.main:app
```

На Windows это может поднять сервер не из той версии Python, и тогда обработка будет падать, например с ошибкой:

`demucs is not installed`

## Что загружать на GitHub

В репозиторий нужно загружать исходники проекта:

- `backend/`
- `frontend/`
- `scripts/`
- `README.md`
- `ИНСТРУКЦИЯ.md`
- `.gitignore`
- `start-app.bat`

Если появятся дополнительные файлы проекта вроде `LICENSE`, `CHANGELOG`, скриншотов или примеров, их тоже можно загружать.

## Что не нужно загружать на GitHub

Не нужно загружать локальные и автоматически созданные файлы:

- `.venv/`
- `frontend/node_modules/`
- `frontend/dist/`
- `storage/projects/`
- `storage/app.db`
- `tools/ffmpeg/`
- `.pytest_cache/`
- `.playwright-cli/`
- временные файлы вроде `tmp-create.json` и `tmp-upload.json`

Это уже исключено через `.gitignore`.

## Структура проекта

```text
backend/      FastAPI, пайплайн, рендер, API
frontend/     интерфейс на React
scripts/      bat/ps1-скрипты для запуска и bootstrap
storage/      локальные проекты и экспорты
tools/        ffmpeg и вспомогательные утилиты
```

## Для GitHub в двух словах

Если хочешь загрузить проект на GitHub без лишней возни:

1. Оставь в папке только исходники
2. Не трогай `.gitignore`
3. Загружай проект целиком, но без `.venv`, `storage/projects`, `node_modules` и `tools/ffmpeg`

Если будешь делать это через `git`, то достаточно обычного:

```bash
git init
git add .
git commit -m "Initial commit"
```

При нормальном `.gitignore` мусор сам не попадёт в репозиторий.
